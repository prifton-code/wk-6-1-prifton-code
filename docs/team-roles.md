## 1. Project Lead / Administrator
Name: [Prifton Mliwa]  
Role in Jira: Administrator  
Responsibilities:
- Create and configure the Jira project (boards, sprints, workflows, permissions).  
- Add team members and assign project roles.  
- Plan sprints and oversee progress using Scrum boards.  
- Review issue updates, verify workflow transitions, and close completed sprints.  
- Coordinate between Developers and QA to ensure timely delivery.  
- Maintain documentation and version control setup (e.g., GitHub/Bitbucket link).  

Jira Access Permissions:
- Full administrative rights within the project.  
- Can manage roles, issues, and reports.

##  2. Developer
Name: [Peter Adebisi]  
Role in Jira: Developer  
Responsibilities:
- Implement assigned features, fixes, and enhancements.  
- Create and update issues (Tasks, Stories, Bugs) in Jira.  
- Push commits and branches referencing Jira issue keys.  
- Collaborate with QA during testing and bug verification.  
- Update issue status from To Do → In Progress → In Review.  
- Participate in sprint planning and daily stand-ups.  

Jira Access Permissions:
- Can create, edit, and transition issues.  
- Can view and comment on all project issues.

##  3. QA Tester
Name: [Lena Wahu]  
Role in Jira: Tester  
Responsibilities:
- Design and execute test cases for all developed features.  
- Capture evidence (screenshots, logs) and attach to Jira issues.  
- Report bugs and link them to related stories or tasks.  
- Verify fixes before issues are moved to Done.  
- Maintain a Test Report or “Evidence Log” in Jira.  
- Support continuous improvement by reporting test results to the Project Lead.  

Jira Access Permissions:
- Can create and edit Bugs.  
- Can comment, attach files, and transition issues for verification.  
- Read-only access to sprint board configuration.
